# 📘 Assignment – Part I: Gradio Interactive EDA Interface

## 📌 Introduction

**Gradio** is a Python library that allows you to build interactive web interfaces for data exploration, visualization, and machine learning models with minimal code.

In this phase, you will use Gradio as a companion to your machine learning notebooks. Instead of presenting static charts, you will transform your analyses into **interactive interfaces** that allow users to explore the data themselves.

The goal is not simply to display existing visualizations, but to:

* Enable filtering and querying of data
* Allow dynamic exploration
* Provide meaningful insights through interaction

Think of this application as a **daily-use tool for a data analyst at PXL Pedals**, designed for quick and efficient data consultation.

---

## 🎯 Objective

Build a **multi-tab Gradio application** that supports interactive data exploration and insight generation.

---

## 🛠️ Requirements

### ✅ Core Requirements

* Create a **Gradio app with multiple tabs**
* Each tab should represent a **specific perspective of the dataset**
* The app must be **interactive**, not static

### 📊 Suggested Data Views

As a group, decide which perspectives are most useful. Examples include:

* Trips analysis
* Bike demand trends
* Station activity
* Time-based usage patterns

---

### ⚙️ Technical Requirements

* The application must run with a **single command**:

  ```bash
  python app.py
  ```
* The app should be **self-contained and standalone**

---

### 📄 Documentation Requirement

Include a **README.md** file in the Gradio folder that explains:

* How to run the application
* What each tab does
* Any required dependencies

---

## 💡 Expectations

Your application should:

* Go beyond static notebook outputs
* Provide meaningful interactivity
* Help users explore and discover insights independently

Focus on usability, clarity, and practical value for data analysis.
